# /dev-docs — 개발 계획 수립

사용자의 작업 요청을 분석하여 구현 전 전략적 계획을 수립합니다.

## 실행 절차

1. **분석**: 요청 범위, 영향 파일, 의존성 파악
2. **계획**: 단계별 구현 전략 수립
3. **문서화**: `docs/active/{task-name}/` 에 3개 문서 생성

### 생성 문서

| 파일 | 내용 |
|------|------|
| `plan.md` | 구현 전략, 단계별 절차, 예상 리스크 |
| `context.md` | 관련 파일, 의존성, 기존 패턴 분석 |
| `tasks.md` | 체크박스 형식의 실행 항목 |

## 원칙

- 코딩 전에 생각하고 계획한다
- 계획은 실행 가능한 수준으로 구체적으로
- 불확실한 부분은 명시하고 대안 제시
