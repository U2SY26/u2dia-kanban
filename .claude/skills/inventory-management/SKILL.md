# Inventory Management — 재고 관리

## 사용 시기
- "재고", "입출고", "발주", "SKU", "창고 관리" 요청 시

## 핵심 기능

### 1. 실시간 재고 동기화
- 멀티 채널 재고 통합 (쿠팡 + 네이버 + 11번가 = 하나의 재고)
- 주문 발생 시 즉시 차감 (모든 채널 동시 반영)
- 안전 재고 경고 (임계값 설정)

### 2. 입출고 관리
- 입고: 발주 → 검수 → 입고 확정 → 재고 반영
- 출고: 주문 확인 → 피킹 → 패킹 → 출고 → 재고 차감
- 반품: 반품 접수 → 검수 → 재입고/폐기

### 3. 자동 발주
- 안전 재고 미달 시 자동 발주 알림
- 리드타임 기반 발주 시점 계산
- 경제적 주문량(EOQ) 제안

### 4. SKU 관리
- 상품별 고유 SKU 체계
- 옵션 조합별 SKU 자동 생성
- 바코드/QR 코드 연동

## 데이터 모델

```
inventory
├── sku (고유 코드)
├── product_id (상품 참조)
├── warehouse_id (창고)
├── quantity (현재 수량)
├── reserved (예약 수량 — 미발송 주문)
├── available (quantity - reserved)
├── safety_stock (안전 재고)
├── reorder_point (재발주점)
└── updated_at (마지막 변경)
```

## 원칙

1. **정합성** — 재고 수량은 항상 정확해야 함 (트랜잭션 보장)
2. **실시간** — 채널 간 재고 동기화 지연 최소화
3. **추적** — 모든 재고 변동에 로그 기록 (누가, 언제, 왜, 얼마나)
4. **알림** — 안전 재고 미달, 과잉 재고, 이상 변동 시 즉시 알림
