# Internal Comms — 팀 소통

## 사용 시기
- "상태 보고", "주간 보고", "뉴스레터", "공지사항", "변경 안내" 작성 요청 시

## 템플릿 유형

### 프로젝트 상태 보고
```
## [프로젝트명] 상태 보고 — YYYY-MM-DD

### 요약
- 전체 진행률: X%
- 이번 주 완료: [항목]
- 다음 주 계획: [항목]
- 차단 이슈: [있으면 기술]

### 상세
[세부 내용]
```

### 변경 안내
```
## 변경 안내 — [제목]

### 무엇이: [변경 내용]
### 왜: [이유]
### 언제: [시점]
### 영향: [영향 범위]
### 조치: [필요한 행동]
```

## 원칙

1. 간결하고 구조화된 작성
2. 수신자 관점에서 필요한 정보 우선
3. 행동 항목(Action Item)은 명확히 분리
4. 긴급도에 따라 톤 조절
